symlink vs hardlink
-------------------

symlink is a short cut for the original file. for symlink it creates new inode. if original file deleted symlink not get the data.

hard link directly point out inode it doesn't care about original file. if original file deleted then also hardlink gets the data. because inode not deleted. if all files deleted related to inode then the inode also deleted.
