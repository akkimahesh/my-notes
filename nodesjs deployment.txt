🏗️ Architecture Overview
Internet
   ↓
Nginx (80/443)
   ↓
PM2 Processes (Different Ports)
   ↓
Node.js Apps
Each environment runs on a different port:

Environment	Port
DEV	3001
QA	3002
PROD	3003
✅ STEP 1 — Install Required Software
SSH into EC2:

sudo apt update -y
sudo apt install nginx -y
sudo apt install certbot python3-certbot-nginx -y
sudo apt install nodejs npm -y
sudo npm install -g pm2
Verify:

nginx -v
node -v
pm2 -v
✅ STEP 2 — Project Structure (Recommended)
/var/www/
    ├── app-dev
    ├── app-qa
    └── app-prod
Create directories:

sudo mkdir -p /var/www/app-dev
sudo mkdir -p /var/www/app-qa
sudo mkdir -p /var/www/app-prod
Clone your repo into each folder.

✅ STEP 3 — Configure Environment Ports
In your Node app (example: server.js):

const port = process.env.PORT || 3000;
app.listen(port, () => {
  console.log(`Server running on port ${port}`);
});
✅ STEP 4 — Start with PM2
Go to each folder:

DEV
cd /var/www/app-dev
npm install
PORT=3001 pm2 start server.js --name app-dev
QA
cd /var/www/app-qa
npm install
PORT=3002 pm2 start server.js --name app-qa
PROD
cd /var/www/app-prod
npm install
PORT=3003 pm2 start server.js --name app-prod
Save PM2:

pm2 save
pm2 startup
✅ STEP 5 — Configure Nginx Reverse Proxy
Create nginx config:

sudo nano /etc/nginx/sites-available/app
🔹 If using Subdomains (Recommended)
Example:

dev.yourdomain.com

qa.yourdomain.com

yourdomain.com (prod)

Paste:

server {
    server_name dev.yourdomain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}

server {
    server_name qa.yourdomain.com;

    location / {
        proxy_pass http://localhost:3002;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}

server {
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:3003;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
    }
}
Enable it:

sudo ln -s /etc/nginx/sites-available/app /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
✅ STEP 6 — Configure SSL (Certbot)
Make sure DNS points to EC2 public IP.

Run:

sudo certbot --nginx -d dev.yourdomain.com -d qa.yourdomain.com -d yourdomain.com
Certbot will:

Automatically configure SSL

Redirect HTTP → HTTPS

Test auto renewal:

sudo certbot renew --dry-run
🎯 Final Setup
Now:

URL	Environment
https://dev.yourdomain.com	DEV
https://qa.yourdomain.com	QA
https://yourdomain.com	PROD

pm2 start ecosystem.config.js
pm2 save

✅ Short Answer
👉 Keep ecosystem.config.js OUTSIDE individual environment folders.

🎯 Best Practice for Your Setup (Same EC2, 3 Environments)
Since you have:

/var/www/
    ├── app-dev
    ├── app-qa
    ├── app-prod
✅ Recommended Structure:
/var/www/
    ├── app-dev/
    ├── app-qa/
    ├── app-prod/
    └── ecosystem.config.js   👈 HERE
🚀 Why Outside?
Because:

✔ It manages multiple applications
✔ It is infrastructure-level config
✔ It’s not part of a single app
✔ Cleaner separation of concerns

Think like this:

Node app → application code

ecosystem file → process manager config (infra)

🔥 How You Run It
From /var/www:

pm2 start ecosystem.config.js
PM2 will look inside each cwd:

cwd: "/var/www/app-dev"
🧠 When Should You Keep It Inside the App Folder?
Only when:

You have single app

OR each environment is on separate server

OR using CI/CD per environment

Example:

app/
 ├── server.js
 ├── package.json
 └── ecosystem.config.js
That is common in Docker or separate EC2 setups.

🏆 Professional Recommendation (For You)
Since you're running:

Dev

QA

Prod

Same EC2

👉 Keep ecosystem file in /var/www/

Clean.
Professional.
Maintainable.
